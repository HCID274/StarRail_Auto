# 三月七助手（March7thAssistant）特性与命令行用法

> 基于实际测试和官方 README 整理，记录自动化调用时需要知道的关键信息。

## 一、两个可执行文件的区别

| 文件 | 用途 | 特点 |
|------|------|------|
| `March7th Launcher.exe` | GUI 版（三月七图标） | 打开图形界面，可带参数自动执行任务 |
| `March7th Assistant.exe` | 终端版（终端图标） | 无 GUI，适合任务计划程序和脚本调用 |

**当前安装版本实测：需要使用 `March7th Launcher.exe <task> -e` 才能正确接收任务参数并启动任务。**

## 二、命令行参数

```bash
# 查看帮助
March7th Launcher.exe -h

# 列出所有可用任务
March7th Launcher.exe -l

# 执行完整运行
March7th Launcher.exe main

# 执行完整运行，完成后自动退出
March7th Launcher.exe main -e

# 执行模拟宇宙，完成后自动退出
March7th Launcher.exe universe -e
```

### 关键参数

| 参数 | 说明 |
|------|------|
| `<task>` | 任务名（见下方任务列表） |
| `-e` | 任务正常完成后自动退出进程（**自动化必加**） |
| `-h` | 查看帮助 |
| `-l` | 列出所有可用任务 |

## 三、踩坑记录

### 1. 需要管理员权限

两个 exe 的 manifest 都嵌入了 `requireAdministrator`，普通权限下 `subprocess.Popen()` 会报错：

```
OSError: [WinError 740] 请求的操作需要提升
```

**解决方案**：不需要在代码里用 `ShellExecuteEx` 提权。Windows 任务计划程序设置"使用最高权限运行"，脚本以管理员权限启动后，子进程继承权限，`subprocess.Popen()` 可直接调用。

### 2. 自动化必须加 `-e` 参数

M7A 默认 `pause_after_success: true`，任务完成后会暂停等待用户确认，不会自动退出。加 `-e` 参数可覆盖此行为，任务完成后自动退出进程。

如果不加 `-e`，也可以在 `config.yaml` 中设置：
```yaml
pause_after_success: false
after_finish: Exit
```

但 `-e` 参数更安全，不会影响手动使用时的行为。

### 3. Launcher vs Assistant

官方本地 README 把命令行参数示例写在 `March7th Launcher.exe` 上，且当前安装版本实测也是：

- `March7th Launcher.exe <task> -e` 可以正常启动任务
- `March7th Assistant.exe <task> -e` 会快速退出，返回 `exit code 2`

因此，本项目正式链路当前统一使用 `March7th Launcher.exe <task> -e`。

## 四、相关配置（config.yaml）

与自动化运行相关的配置项：

```yaml
# 成功后暂停（-e 参数可覆盖）
pause_after_success: true

# 失败后直接退出
exit_after_failure: false

# 任务完成后操作
# 可选值：None / Exit / Loop / Shutdown / Sleep / Hibernate / Restart / Logoff / TurnOffDisplay / RunScript
after_finish: None

# 游戏路径
game_path: D:\...\StarRail.exe

# 启动游戏超时（分钟）
start_game_timeout: 10
```
