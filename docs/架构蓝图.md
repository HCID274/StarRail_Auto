# 星穹铁道自动化 - 架构蓝图

> 本文档是后续 AI 编程的唯一指导蓝图。

## 一、项目目标

在日本环境下，全自动完成崩坏：星穹铁道的日常任务。电脑不休眠、保持登录不锁屏，脚本按计划执行，跑完即退出。

## 二、完整执行流程

```
Windows 任务计划程序（调度）
  │
  ▼
m7a_runner.py（入口）
  │
  ├─ 1. 以管理员权限进入正式链路
  ├─ 2. 启动 UU 加速器（如未运行）
  ├─ 3. 将 UU 窗口置顶到前台
  ├─ 4. 若 UU 已在运行，先短轮询 `templates/uu_03.png`
  ├─ 5. 若 `uu_03.png` 已存在，则直接复用当前加速状态
  ├─ 6. 若 `uu_03.png` 不存在，再执行完整识图链路
  ├─ 7. 先等待 3 秒，再以 0.25 秒频率轮询 `templates/uu_01.png`
  ├─ 8. 识别成功后移动鼠标到中心
  ├─ 9. 等待 0.5 秒
  ├─ 10. 以 0.25 秒频率轮询 `templates/uu_02.png`，识别后点击
  ├─ 11. 等待加速生效并识别 `templates/uu_03.png` 作为确认
  │
  ├─ 12. 启动 March7th Launcher.exe，传入任务参数和 `-e`
  ├─ 13. 轮询确认 `StarRail.exe` 或游戏窗口已真正出现
  ├─ 14. 看门狗监控（CPU 空闲 / 日志心跳 / 硬超时）
  └─ 15. 任务结束，记录日志，进程退出
```

## 三、任务时间表

| 时间  | 任务         | 参数       | 超时    |
|-------|-------------|-----------|---------|
| 23:30 | 模拟宇宙     | universe  | 7200s   |
| 07:00 | 完整运行     | main      | 1800s   |

任务之间留 5 分钟缓冲，避免前一个超时未退出时后一个抢鼠标。

## 五、模块职责

### 1. `m7a_runner.py` — 核心执行器

- 命令行入口：`python m7a_runner.py <task> [--timeout SEC]`
- 调用 `uu_accel.py` 执行正式 UU 启动链路
- 启动 `March7th Launcher.exe` 并传入任务名和 `-e`
- 启动后先确认游戏进程或游戏窗口真的出现，再进入常规看门狗
- 看门狗监控：
  - **硬超时**：到时间强制 kill 进程树
  - **CPU 空闲检测**：连续 15 分钟 CPU < 2% → 判定卡死 → kill
  - **日志心跳检测**：M7A 日志 10 分钟未更新 → 判定卡死 → kill
  - **启动宽限期**：60s 内不做卡死判定
- 统一退出码：
  - `0`：正常结束
  - `10`：UU 链路失败
  - `20`：M7A 启动器拉起失败
  - `21`：游戏未在启动确认时间内出现
  - `22`：M7A 非零退出
  - `30/31/32`：看门狗因硬超时 / CPU 空闲 / 日志心跳停滞触发
- 日志输出到 `logs/YYYY-MM-DD.log`（用文件夹装起来，不散落到根目录）

### 2. `uu_accel.py` — UU 正式启动链路

- 检测 UU 加速器进程是否已运行，未运行则启动
- GUI 操作前要求当前进程已提权
- 将 UU 窗口置顶到前台
- 若 UU 已在运行，先短轮询 `uu_03.png` 判断是否已经处于可复用的加速完成状态
- 若 `uu_03.png` 已存在，则直接复用，不再点击 `uu_01.png` / `uu_02.png`
- 若 `uu_03.png` 不存在，再继续完整启动链路
- 先等待 3 秒，再以 0.25 秒频率轮询 `uu_01.png`
- 识别成功后将鼠标移动到中心
- 等待 0.5 秒
- 以 0.25 秒频率轮询 `uu_02.png` 并点击
- 等待加速生效后识别 `uu_03.png` 作为连接确认
- 失败时截图存入 `debug/`

**当前阶段原则**：每完成一步，就直接并入 `ensure_uu_connected()` 正式链路；不保留平行测试脚本或一次性测试分支。

### 3. `run_elevated.py` — 统一提权入口

- 手动测试 GUI 自动化时，统一通过该脚本拉起管理员进程
- 入口形式：`uv run python run_elevated.py <script.py> [args...]`
- 作用边界：只负责提权和转发命令，不承载业务逻辑

### 4. `templates/` — 图像模板

- 手动截取 UU 加速器界面中的关键元素
- 在当前环境（2K 分辨率 + 125% 缩放）下截取
- 当前正式链路使用 `uu_01.png`、`uu_02.png`、`uu_03.png`

## 六、技术选型

| 用途           | 库                        | 理由                           |
|---------------|--------------------------|-------------------------------|
| 图像识别       | pyautogui + pillow + opencv-python | 截图、匹配、点击链路直接可用 |
| 键鼠操作       | pyautogui                 | click/moveTo 简单够用           |
| 进程管理       | psutil                    | 进程树 kill、CPU 监控            |
| 日志           | logging（标准库）           | 零额外依赖，按天写文件             |

## 七、依赖管理

使用 **uv** 管理 Python 依赖和虚拟环境。

```toml
# pyproject.toml 核心依赖
[project]
dependencies = [
    "opencv-python>=4.9",
    "pillow>=12.2.0",
    "psutil>=5.9",
    "pyautogui>=0.9",
]
```

常用命令：
```bash
uv init          # 初始化项目
uv add psutil pyautogui pillow opencv-python  # 添加依赖
uv run python run_elevated.py uu_accel.py     # 手动测试 UU 正式链路
uv run python run_elevated.py m7a_runner.py main --timeout 1800  # 运行完整链路
```

## 九、Windows 任务计划配置要点

- `LogonType` 使用 `Interactive`，只在用户已登录时运行
- 使用最高权限运行
- 动作统一调用 `powershell.exe -File scripts/run_scheduled_task.ps1 ...`
- 脚本内部会 `Set-Location` 到项目目录，再执行 `uv run python m7a_runner.py ...`
- 取消交流电源限制
- 任务级超时设为脚本超时的 1.5 倍（兜底）

## 十、注意事项

1. 电脑必须保持登录且桌面可交互，否则 GUI 自动化会失败
2. 显示器休眠不影响运行（截屏取的是 GPU 帧缓冲）
3. 星穹铁道客户端关闭"空闲时自动关闭"选项
4. 图像识别失败时自动截图到 `debug/`，方便事后排查
5. 125% 缩放下截取模板时，确保使用实际像素截图（非逻辑像素）
